  `2025-10-0102-Article_Title.md`

- 先看整頁版面，再閱讀頁面內容，最後才把完整內容寫入對應 markdown 檔案

- 不要改動其他文章檔案
- 若你發現某一頁其實是下一篇文章，必須停在本篇結束處，不要混入下一篇

## 5. 關於各區塊的規則

- `by Pamela Osment` 這種作者列，保留成純文字，不要改成 `Author:`
- 原文小標維持英文原標，不要翻譯
- `Word Bank`、`Word Bank Boost`、`Word Bank Review` 的格式固定為：
  `plentiful (adj.) 多的，豐富的`
  下一行直接放例句
  不要加 `Example:`
  每個單字之間空一行
- `More Information` 通常只有詞條，不一定有例句；若原頁沒有例句，就不要自作主張補
- `Focus on Phrases`、`Phrasal Verbs/Idioms` 每行一個片語
- `Usage Tip` 是文法區
- 若文章跨兩天，兩頁或三頁裡可能有 2 個 `Usage Tip`，請都收進同一個 `## Usage Tip` 區塊，分成 `### Tip 1`、`### Tip 2`
- `Talk About It` 一定放在文章最後一個主要學習區塊
- 若該篇有 `Activities`，就放在最後
- 若某篇沒有某個區塊，就不要硬加
- 若有 `Key Points` 填空題，請整理成完整可讀的重點句，而不是保留底線空格

## 6. OCR 與整理原則

- 先做版面辨識，再做文字擷取；不要只依賴 OCR 順序
- 對相片來源來說，先判斷 `日期 / 印刷頁碼 / 標題 / 區塊位置`，再抽文字
- 先辨認大標與區塊骨架，例如 `Talk About It`、`Test Yourself`、`Usage Tip`、`Key Point`、`Word Bank`
- 若同一張圖同時拍到跨頁，必須先分清左頁與右頁各自屬於哪一篇、哪一區塊
- 若 mapping 已註明哪一側接課文、哪一側是語法區或練習區，不可用 OCR 內容覆蓋這個判斷
- 先看整頁版面，再視需要裁切、放大局部確認
- 若文字模糊，可根據上下文、固定欄位格式、常見詞彙做保守修正
- 但不要亂猜；不確定時寧可保守
- 要特別留意單複數、固定片語、專有名詞、人名、品牌名
- 若相片有反光或歪斜，先以可辨識區塊為主，必要時多次裁切查看
- 不要把下一篇文章的內容混入本篇

## 7. 寫檔規則

- 只更新我指定的目標檔案
- 直接把完整內容寫入現有空檔
- 保持 markdown 結構清楚、可讀
- 不要加入多餘說明、不要寫你的分析、不要在檔案中留下 `OCR uncertain` 之類的註記，除非我要求

## 8. 回報方式

- 完成後只簡短告訴我：
  - 已更新哪個檔案
  - 這篇用了哪些頁（例如 p20, p21, p22）

- 不要冗長總結

## 9. 重要判斷基準

- 一律以雜誌頁面上印刷的頁碼為準
- 不以 PDF 的內部頁碼為準
- 若我說「8/8, 8/9 是 p20, p21, p22」，你必須理解這是雜誌印刷頁碼，不是 PDF 第 20 到 22 頁

## 10. 你開始工作時的流程

- 先查看目標目錄中對應的檔案
